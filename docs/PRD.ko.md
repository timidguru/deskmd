# DeskMD PRD

[English](./PRD.md) | [한국어](./PRD.ko.md)

## 1. 개요

DeskMD는 MacBook에서 별도 앱으로 실행되는 경량 마크다운 에디터다. 사용자는 브라우저를 직접 열지 않고 `DeskMD.app`을 실행해 마크다운 문서를 작성, 미리보기, 저장, HTML 내보내기 할 수 있다.

현재 제품은 로컬 macOS 앱 번들 안에 웹 기반 편집 UI를 포함하고, 네이티브 `WKWebView`로 실행한다.

## 2. 목표

- macOS에서 앱처럼 실행되는 마크다운 작성 환경을 제공한다.
- 좌우 분할 화면으로 작성 내용과 미리보기를 즉시 확인한다.
- 별도 서버나 패키지 설치 없이 로컬 앱 번들로 실행한다.
- `.md`, `.markdown`, `.txt` 파일을 열고 `.md` 또는 `.html`로 저장한다.
- 작성 중인 내용을 로컬 자동 저장으로 보존한다.

## 3. 비목표

- 전문 IDE 수준의 프로젝트/폴더 관리 기능.
- 다중 탭 문서 편집.
- Git 연동, 동기화, 클라우드 저장.
- App Store 배포, 개발자 인증서 기반 공증.
- 완전한 CommonMark/GFM 호환 렌더러의 자체 구현.

## 4. 사용자

- MacBook에서 빠르게 마크다운 문서를 작성하려는 개인 사용자.
- README, 메모, 블로그 초안, 간단한 기술 문서를 작성하는 사용자.
- 웹 브라우저 탭이 아니라 독립 앱으로 편집기를 실행하고 싶은 사용자.

## 5. 주요 사용 시나리오

### 5.1 새 문서 작성

1. 사용자가 `DeskMD.app`을 실행한다.
2. 기본 문서 또는 마지막 자동 저장본이 열린다.
3. 왼쪽 편집 영역에 마크다운을 입력한다.
4. 오른쪽 미리보기 영역이 즉시 갱신된다.

### 5.2 기존 파일 열기

1. 사용자가 `열기` 버튼을 누른다.
2. macOS 파일 선택창이 열린다.
3. `.md`, `.markdown`, `.txt` 파일을 선택한다.
4. 파일 내용이 편집 영역에 로드되고 미리보기가 갱신된다.

### 5.3 마크다운 저장

1. 사용자가 `MD 저장` 버튼 또는 `Cmd+S`를 누른다.
2. macOS 저장 패널에서 저장 위치를 선택한다. 기존 문서를 연 상태라면 기본 위치는 해당 문서가 있던 폴더다.
3. 현재 문서가 `.md` 파일로 저장된다.
4. 상태 텍스트에 저장 완료 메시지가 표시된다.

### 5.4 HTML 내보내기

1. 사용자가 `HTML 저장` 버튼을 누른다.
2. 현재 미리보기 HTML이 독립 HTML 문서로 생성된다.
3. 브라우저나 다른 도구에서 열 수 있는 `.html` 파일로 저장된다.

## 7. 비기능 요구사항

| ID | 요구사항 | 기준 |
| --- | --- | --- |
| NFR-001 | 실행 편의성 | 빌드 산출물 또는 설치 대상 앱에서 실행 가능 |
| NFR-002 | 설치 부담 | Node/Electron 서버 없이 실행 |
| NFR-003 | 성능 | 일반 문서 편집 시 입력 지연 없이 미리보기 갱신 |
| NFR-004 | 보안 | 렌더링 HTML에서 스크립트성 요소를 제거 |
| NFR-005 | 오프라인 | 앱 UI, `marked`, `DOMPurify`는 인터넷 없이 동작 |
| NFR-006 | 유지보수 | 빌드 스크립트 하나로 앱 번들을 재생성 |

## 8. 현재 제약

- 앱 번들에 포함된 `marked 18.0.0`과 `DOMPurify 3.3.3`을 기본 렌더러로 사용한다.
- 인터넷 연결이 있으면 npm registry에서 최신 버전 여부만 확인한다.
- 저장은 네이티브 macOS 저장 패널로 처리한다.
- 앱은 ad-hoc 서명이며 Apple 개발자 계정 공증은 되어 있지 않다.
- 자동 저장은 앱 내부 WebView의 `localStorage`에 저장된다.

## 9. 향후 개선 후보

- 업데이트 가능 알림 UX 개선.
- 기존 파일 덮어쓰기와 `Save As` 구분.
- 최근 문서 목록.
- 문서 제목/파일명 표시 개선.
- 다크 모드 지원.
- 앱 아이콘 추가.
- 공증 배포용 빌드 파이프라인 추가.
